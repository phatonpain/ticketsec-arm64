## Calibration Decision Log

- **Method:** Top-label Expected Calibration Error (ECE) with 10 equal-width bins
  and multiclass Brier score, computed directly from the INT8 ONNX probability
  outputs on the held-out test set.
- **Result:** ECE = 0.3946, Brier = 0.3194, assessment = `UNDERCONFIDENT`.
- **Interpretation:** The `class_weight="balanced"` logistic-regression pipeline
  is under-confident on the easy, synthetic dataset. Because the dashboard uses
  argmax classification (not a confidence threshold), this is a documented
  finding rather than a blocker.
- **Future action:** If confidence thresholds are introduced, apply temperature
  scaling or isotonic recalibration and re-run `model/check_calibration.py`.

## Known Limitations

1. The dataset is synthetic and highly separable; metrics are demo-grade.
2. The winning ablation candidate (`C2_char3-5_word1-2_LR_C2.0`) cannot be
   exported to ONNX because `skl2onnx` does not support `analyzer="char_wb"`.
   The deployed artifact uses the best ONNX-exportable candidate.
3. Graviton latency numbers in `model/latency_t4g_micro.json` are from a local
   emulated endpoint; live AWS measurements are pending.
4. Calibration is under-confident; confidence values should not drive automated
   decisions without recalibration.
